# Myrmec Use Cases & Strategic "Why"
**Role:** Conversion & Credibility

## 1. The "No-Developer" Use Cases
- **Case 1: Self-Healing Repositories.** Schedule an agent to scan for vulnerabilities and auto-generate PRs based on internal security standards.
- **Case 2: Legacy Transformation.** Orchestrate a workflow to migrate Java 8 monoliths to Spring Boot 3.4 microservices using pre-defined Sutras.
- **Case 3: Architectural Audit.** Agents act as "24/7 Tech Leads," reviewing all code against the company's Knowledge Source before it reaches a human.

## 2. The "Why Myrmec" Section
- **Institutional Memory:** Don't let AI knowledge leave when a developer quits. Capture prompts and logic in Agent Profiles.
- **Shadow AI Mitigation:** Provide a secure, governed environment so employees stop using unmanaged personal AI accounts. Set token limits at various levels.
- **Vendor Agnostic:** Swap models (DeepSeek to Llama) with one click in the Admin Panel without breaking your project workflows.
